> 最終更新: 2026-01-21

## エグゼクティブサマリー

Obsidian AIプラグイン市場は**成熟期**に入り、主要3軸（チャット/検索、生成/編集、エディタ連動）で競合が固まっている。**ローカルLLM対応**と**プライバシー重視**が標準化し、差別化は**インライン体験**と**特化型プロンプト**に移行中。**開発者向け特化**は未開拓で、GitHub連携とコード文脈最適化が最大の機会領域。

---

## 需要とユーザー規模（見込み）

### 想定ユーザー層
- **コア層**: Obsidianのパワーユーザー（知識管理・研究・開発者）
- **隣接層**: AIノート整理や執筆支援を求める一般ユーザー
- **本プロジェクトの狙い**: ローカル主権・拡張性を重視する層

### 規模の考え方（概算）
- **総潜在**: Obsidianユーザー全体（公開統計が少ないため幅で推定）
- **AI積極層**: プラグインでAIを使う層は全体の一部（体感で数％〜十数％）
- **Open Source志向**: プライバシー/主権重視の層はさらに限定的だが熱量が高い

### 目安としての現実的レンジ
- **初期目標**: 数百〜数千インストール
- **中期目標**: 5,000〜20,000インストール
- **上限想定**: 30,000〜100,000インストール（大型プラグイン級）

### 需要ドライバー
- **ローカル主権**: データを外部に出したくないニーズ
- **編集自動化**: 文章修正・ノート整理の省力化
- **拡張性**: 自分のワークフローに合わせられるAI
- **低摩擦**: チャットだけで完結する操作体験

## 市場ギャップと機会

### 未開拓領域

1. **コードブロック文脈最適化** ⚡
   - 現状: Copilot/Text Generatorはコード生成可能だが文脈認識が弱い
   - 機会: 言語・依存関係・プロジェクト構造を理解した提案

2. **インライン提案** ⚡
   - 現状: Infio Copilotのみ実装
   - 機会: VS Code Copilot風の洗練されたUX

3. **GitHub文脈連携** ⚡⚡
   - 現状: なし
   - 機会: Issue/PR/コミット履歴の自動統合

4. **開発者向けプロンプト** ⚡
   - 現状: 汎用プロンプトのみ
   - 機会: 言語別テンプレート、コードレビュー、デバッグ支援

### 差別化の方向性

**「開発者向けAIチャット」を明示的に狙う**
- コード補完・説明・リライトをノート内で完結
- GitHub Copilot前提のプロンプト設計
- インライン提案で明確な差別化（Infio Copilotとの競合に注意）
- GitHub連携で独自性（後追いでも強い差別化要素）

---

## MVPへの示唆

### Phase 1: 基本チャット + ストリーミング
- Smart Chat/Copilotの成功パターン参考
- サイドバー or Codeblock統合
- OpenAI API完全対応

### Phase 2: インライン提案
- Infio Copilot以外に競合少ない
- CodeMirror装飾でゴーストテキスト
- GitHub Copilot風UX

### Phase 3: GitHub連携
- Issue/PR/コミット履歴統合
- リポジトリコンテキスト自動取得
- 開発者ワークフロー最適化

## 技術スタック標準

### 共通選択
- **言語**: TypeScript 95%+
- **ビルド**: ESBuild/Rollup
- **テスト**: Jest
- **UI**: Obsidian Native + 必要に応じてReact
- **API**: OpenAI互換エンドポイント

### 必須要件
- ✅ ローカルLLM対応（市場標準）
- ✅ ストリーミング対応（必須UX）
- ✅ プライバシー重視（デフォルトローカル）
